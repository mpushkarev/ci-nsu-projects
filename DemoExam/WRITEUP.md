# Как сдать демку, заметки на полях

> [!CAUTION]
> Просто заметки для себя, не руководство к действию :D

## Блок 1С

1. Открываем 1С и создаем пустой проект (в соответствии с требованиями к названию).

2. На второй странице справки `getback-mywpf` инфа подробнее.\
   Если кратко, то переходим в удобную директорию и выполняем `getback-mywpf get-1c`.

3. После в 1С открываем режим Конфигурации, переходим в **<kbd> Конфигурация</kbd> ->**\
   **<kbd>Сравнить, объединить с конфигурацией из файла...</kbd>**, выбираем наш файл.

4. Далее надо почистить логи, заходим в **<kbd>Администрирование</kbd> -> <kbd>Настройка журнала регистрации</kbd>**.\
   Выбираем **<kbd>Не регистрировать</kbd> -> <kbd>Сократить</kbd>**, следующий день от текущего.

5. Переходим в режим Предприятия (или режим отладки), тыкаем по ссылке **<u>Пользователи</u>** в верхнем левом углу.\
   Создаем несколько демо-пользователей, например:

   | Логин | Пароль | Дата входа     | Активность | Роль          |
   |-------|--------|----------------|------------|---------------|
   | admin | 1      | -              | Да         | Администратор |
   | user1 | 1      | -              | Да         | Пользователь  |
   | user2 | 1      | -              | Нет        | Пользователь  |
   | user3 | 1      | > месяца назад | Да         | Пользователь  |

6. Проверяем, чтобы все работало как надо. На этом блок 1С закончен.

## Блок основного проекта

Можно разделить на несколько частей, начнем с подготовки данных.

### Подготовка и нормализация данных

1. Во-первых, надо все данные соединить в одну таблицу на разных страницах для удобства.

2. После надо привести все к 3НФ, можно ориентироваться на схему Артема.

   - `Role` (Роли пользователей):
     - `Id` (int),
     - `RoleName` (string).
   - `Supplier` (Поставщики):
     - `Id` (int),
     - `SupplierName` (string).
   - `Manufacturer` (Производители):
     - `Id` (int),
     - `ManufacturerName` (string).
   - `ProductType` (Типы товаров):
     - `Id` (int),
     - `TypeName` (string).
   - `OrderStatus` (Статусы заказов):
     - `Id` (int),
     - `StatusName` (string).
   - `PickupPoint` (Пункты выдачи):
     - `Id` (int),
     - `Address` (string).
   - `User` (Пользователи):
     - `Id` (int),
     - `RoleId` (int?),
     - `Fullname` (string),
     - `Login` (string),
     - `Password` (string).
   - `Product` (Товары):
     - `Id` (int),
     - `Article` (string),
     - `Price` (decimal?),
     - `SupplierId` (int?),
     - `ManufacturerId` (int?),
     - `ProductTypeId` (int?),
     - `ProductCategoryId` (int?),
     - `Discount` (int?),
     - `UnitOfMeasure` (string),
     - `Amount` (int?),
     - `Description` (string),
     - `Photo` (string).
   - `Order` (Заказы):
     - `Id` (int),
     - `OrderDate` (DateOnly?),
     - `DeliveryDate` (DateOnly?),
     - `PickupPointId` (int?),
     - `UserId` (int?),
     - `Code` (string),
     - `OrderStatusId` (int?).
   - `OrderDetails` (Состав заказа):
     - `Id` (int),
     - `OrderId` (int?),
     - `ProductId` (int?),
     - `Amount` (int?).

   При анализе можно выделять желтым то, что FK, красным шрифтом то, что надо разбить.

3. Лайфхаки нормализации

   - Функция `ВПР`. Создаем столбец рядом с искомым значением.
     В другой таблице временно перемещаем id вправо, пишем примерно это:
     `=ВПР(H2;'название-таблицы'!$A$1:$B$3;2;0)` (не забываем про `$`!),
     где H2 соседний столбец со значением типа `Администратор`, а
     диапазон другой таблицы указывает примерно на это:
     `Администратор | 1`. Таким образом получаем цифру id.
     Множим эту формулу на столбец до конца, копируем только итоговый текст.

   - Для выделения до конца столбца(-ов) выделяем первую строку и 
     **<kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>стрелка вниз</kbd>**.

   - Для формирования списка уникальных значений достаточно выделить все 
     данные столбца способом выше, скопировать их в другую таблицу и
     тут же выполнить **<kbd>Данные</kbd> -> <kbd>Удалить дубликаты</kbd>**.

4. После наведения порядка выделяем столбцы, которые отличаются от схемы,
   и переходим к следующему разделу.

### Создание БД и импорт данных

Подробная информация есть на 3-7 страницах справки `getback-mywpf`.

Кратко: выполняем `getback-mywpf get-db -s СЕРВЕР -d БАЗА_ДАННЫХ`.
При этом саму БД можно вручную не создавать, создастся сама.
После переходим в модели и редактируем их если требуется.

В корне проекта создаем файл `Cookies.cs`, где прописываем
`public static User? LoggedUser { get; set; }`.

Выполняем миграцию по моделям: `getback-mywpf get-db -s СЕРВЕР -d БАЗА_ДАННЫХ -u`. В БД создастся все, что нужно.

Сносим папку миграций и таблицу миграций из бд.

Переходим в MSSMS и начинаем *<s>страдать</s>* переносить данные.
Во избежание конфликтов надо делать в следующем порядке:

1. Роли
2. Пользователи
3. Точки заказа
4. Статусы заказа
5. *(Прочие доп. свойства заказа, если есть)*
6. Заказ
7. Производитель
8. Поставщик
9. Тип продукта
10. Категория продукта
11. *(Прочие доп. свойства продукта, если есть)*
12. Продукт
13. Детали заказа

Обусловлено зависимостями от связей. Если начать не в том порядке, может
сбиться счетчик ключа при ошибке вставки (он щелкает каждый раз, вне
зависимости от успеха операции). Лучше не создавать себе проблем.

Для вставки данных можно **<kbd>ПКМ по таблице</kbd> -> <kbd>Изменить первые 200 строк</kbd>**. Там тыкнуть на стрелочку слева от строки и вставить скопированные данные из таблицы (должны строго соответствовать схеме!). Заголовки копировать не нужно!

В процессе вставки данных главное не дышать, иначе детище Microslop приляжет отдохнуть на пару пинут (в лучшем случае).

### Сборка приложения

Импортируем `.xaml` окна (`getback-mywpf get-wpf`). Подробная информация на 8 странице справки.

Приводим все в порядок, фиксим возникшие ошибки. Перемещаем изображения в `...bin/Images`. Для нужных поделей переопределяем `toString` (достаточно начать писать *public over...*, автодополнение допишет).

На этом этапе все проверить. Обязательно залезть в каждый угол приложения, чтобы нигде не косякнуть!

### Добавление ролей

В соответствии с заданием адаптировать доступы под роли.
Делается достаточно интуитивно и понятно, нет смысла пояснять.

После внимательно проверить соответствие описанному в задании!

### Отчеты

1. Экспорт схемы из MS:\
   Никакого рокетсайнса, тупо скрин в ворд и сохранить в .pdf.

2. Скрипт БД:\
   Тыц **<kbd>ПКМ по БД</kbd> -> <kbd>Задачи</kbd> -> <kbd>Сформировать скрипты...</kbd>**\
   Там выбираем <kbd>Сохранить как файл скрипта</kbd>.\
   Сверху в <kbd>Дополнительно</kbd> выбрать тип данных <kbd>схема и данные</kbd>.\
   Молиться, чтобы поделие Microslop не зависло.

### Доработка приложения

После завершения базовых вещей можно перейти к доработке приложения в оставшееся до конца экзамена время. Как минимум пофиксить некоторые баги.

Полезные советы:

1. Иконка. Проще всего загрузить в корень приложения и тупо расставить у каждого окна `Icon="..\icon.png"`. ВАЖНО! В свойствах иконки указать *собирать как ресурс*!

2. Шрифт. Аналогично, для каждого окна указать `FontFamily="Times New Roman"`, где в кавычках нужный шрифт.
