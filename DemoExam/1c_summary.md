# 1С по шагам

1. Создать инф. базу с названием по принципу *"ДЭ2026_ФамИО"*.

2. Первый шаг: Роли

   - В `Дерево конфигурации > Перечисления` создать объект `Роли`.
   - В разделе `Данные` добавить два значения - **"Администратор"** и **"Пользователь"**.

3. Второй шаг: Справочники

   - В `Дерево конфигурации > Справочники` создать объект `Пользователи`.

   - В разделе `Данные > Стандартные реквизиты` реквизиту `Наименование` задать **Синоним "Логин"**.

   - Далее добавляем следующие реквизиты с соответствующими параметрами:

	 - **Пароль**: `Проверка заполнения` = `Выдавать ошибку`
	 - **КоличествоПопыток**: `Тип` = `Число`
	 - **ДатаВхода**: `Тип` = `Дата (Дата и время)`
	 - **Активность**: `Тип` = `Булево`
	 - **Роль**: `Тип` = `ПеречислениеСсылка.Роли`

4. Третий шаг: Пользователи

   - **Для админа:**

	 - Переходим в раздел `Формы` и добавляем Форму с типом `Форма списка справочника`.

	 - `Далее >`, ставим галочки напротив всех добавленных параметров.

   - **Для пользователя:**

	 - В `Дерево конфигурации > Общие > Общие формы` создаем Форму *"ФормаПользователя"*.
	 
	 - Добавляем текст с помощью `Добавить > Декорация - Надпись` и указываем параметр `Заголовок` как *"Добро пожаловать, это рабочий стол пользователя"*.

   - На этом этапе запускаем Конфигурацию в режиме Отладки и добавляем следующих пользователей:

	 | Логин | Пароль | Дата входа     | Активность | Роль          |
	 |-------|--------|----------------|------------|---------------|
	 | Admin | Admin  | -              | Да         | Администратор |
	 | user1 | 1      | -              | Да         | Пользователь  |
	 | user2 | 2      | -              | Нет        | Пользователь  |
	 | user3 | 3      | > месяца назад | Да         | Пользователь  |

5. Четвертый шаг: Смена пароля

   - В `Дерево конфигурации > Общие > Общие формы` создаем Форму *"ФормаСменыПароля"*.

   - Далее добавляем следующие реквизиты и перетаскиваем влево (на форму):
     
	 - ТекущийПароль
	 - НовыйПароль
	 - ПодтверждениеНовогоПароля

   - У формы в свойствах указываем `ПоложениеКоманднойПанели` = `Нет`.

   - Далее добавляем следующие опции:
     
	 - Реквизит **Роль**: `Тип` = `ПеречислениеСсылка.Роли`
	 - Параметр **ПараметрПользователь**:
	   1) `Тип` = `СправочникСсылка.Пользователи`
	   2) `Ключевой параметр` = `Да`
	 - Команда **ИзменитьПароль**: перетащить на форму.

   - В свойствах Команды нажимаем знакок лупы и выбираем третий пункт (`Создать на клиенте и процедуру на сервере`).

     Откроется окно с кодом, нужно реализовать [следующую](./СменаПароля1.1c) функциональность.

	 Далее в верхней панели в списке процедур добавим `ПриОтрытии`, выбираем третий пункт (`Создать на клиенте и процедуру на сервере`).

	 ТУТ ПОТОМ АДЕКВАТНО КОД СДЕЛАТЬ В ОТДЕЛЬНОМ ФАЙЛЕ!

   - Проверить синтаксические ошибки можно по `Ctrl + F7` или спец. кнопке.

6. Пятый шаг: Авторизация

   - В `Дерево конфигурации > Общие > Общие формы` создаем Форму *"ФормаАвторизации"*.
   
   - У формы в свойствах указываем `ПоложениеКоманднойПанели` = `Нет`.
   
   - Далее добавляем следующие реквизиты с соответствующими параметрами:

	 - **Логин**: `Проверка заполнения` = `Выдавать ошибку`
	 - **Пароль**: `Проверка заполнения` = `Выдавать ошибку`
	 - **ПользовательНайден**: `Тип` = `СправочникСсылка.Пользователи`

   - Добавить Команду **Войти** и перетащить все на форму.

   - В свойствах Команды нажимаем знакок лупы и выбираем третий пункт (`Создать на клиенте и процедуру на сервере`).

   - **код потом**

   - Чтобы форма открывалась при запуске приложения:
     1) ПКМ по корню конфигурации и `Открыть модуль приложения`
	 2) В списке процедур выбрать `ПриНачалеРаботыСистемы` и прописать:
	    ```1c
		ОткрытьФорму("ОбщаяФорма.ФормаАвторизации");
		```
